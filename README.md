# nixos-config

My personal NixOS configuration, built as a flake with home-manager. Single host: `nixos`.

## Layout

| File | Purpose |
| --- | --- |
| `flake.nix` | Inputs (nixpkgs 26.05, home-manager, helium, antigravity-nix) and the `nixos` system output |
| `flake.lock` | Pinned versions of all inputs |
| `configuration.nix` | System entry point: imports `system/*`, plus the user account and `stateVersion` |
| `hardware-configuration.nix` | Machine-specific filesystems and kernel modules (generated by `nixos-generate-config`) |
| `system/` | System modules, one topic per file (boot, graphics, desktop, networking, audio, locale, packages, nix, zapret) |
| `home.nix` | home-manager entry point: imports `home/*` plus the home basics |
| `home/` | home-manager modules, one topic per file (neovim, shell, ghostty, git, ssh, packages, mangohud) |

## Highlights

- **Desktop:** GNOME on Wayland (GDM), with a trimmed default app set and several GNOME extensions managed via home-manager.
- **Graphics:** AMD iGPU + Nvidia dGPU with PRIME render offload (`nvidia-offload` command). Bus IDs are set in `configuration.nix`.
- **Networking/privacy:** dnscrypt-proxy as the local resolver (NetworkManager DNS disabled), plus zapret for DPI bypass. Tailscale enabled.
- **Audio:** PipeWire (Pulse/ALSA compat), PulseAudio disabled.
- **Locale:** US English UI with Turkish regional formats; `tr` keyboard layout.
- **Terminal:** Ghostty with the Cyberdream palette and split/tab keybinds (`home/ghostty.nix`).
- **Shell:** Bash with eza/zoxide/fzf, plus helper aliases (see below).
- **Nix:** flakes + nix-command enabled, store auto-optimise, weekly GC keeping 30 days.

## Usage

These aliases are defined in `home.nix`:

```sh
rebuild   # sudo nixos-rebuild switch --flake /etc/nixos#nixos
update    # nix flake update + rebuild
garbage   # nix-collect-garbage -d
conf      # edit configuration.nix
home      # edit home.nix
flake     # edit flake.nix
```

Or directly:

```sh
sudo nixos-rebuild switch --flake /etc/nixos#nixos
```

### Network switches

Defined in `system/zapret.nix`, passwordless and hotkey-bindable:

```sh
zapret-toggle   # turn the DPI bypass on/off (DNS untouched)
net-portal      # public-wifi / captive-portal mode + panic button:
                # stops zapret + dnscrypt + tailscale and uses the network's
                # plain DNS so a login page can load
net-secure      # restore the secure stack (dnscrypt + zapret + tailscale)
```

Public-wifi login flow: connect to the wifi → `net-portal` → log in on the portal page → `net-secure`.

## DPI bypass & DNS

Turkish ISPs block Discord/YouTube/etc. two ways, handled separately:

- **DNS poisoning** (e.g. `discord.com` → a sinkhole IP) is defeated by `dnscrypt-proxy`, a local encrypted resolver on `127.0.0.1:53` (`system/networking.nix`). NetworkManager's own DNS is disabled so it can't reinstate the ISP resolvers.
- **SNI/DPI blocking** is defeated by `zapret`, with a single TCP/443 `multisplit` profile (`system/zapret.nix`).

**Discord voice/screenshare on the desktop app (Vesktop) does not work** and can't be fixed here: the ISP blackholes voice UDP, and the desktop client's voice engine is UDP-only with no TCP fallback. The Discord **web app** works because browser WebRTC falls back to TURN-over-TCP/443. Use the web app for voice/screenshare; Vesktop is fine for everything else. The only fix for the desktop app would be tunneling through a node outside the block (e.g. a Tailscale exit node abroad), which isn't set up.

## Notes

- Neovim config is **not** in this repo. It is its own git repo living directly at `~/.config/nvim`; `home/neovim.nix` only installs the `neovim` package. Edits there are live (no rebuild needed).
- Wallpapers (`day.jxl`, `night.jxl`) are kept locally and used by GNOME, but are gitignored to keep the repo lean.
- `hardware-configuration.nix` is specific to this machine; regenerate with `nixos-generate-config` on a new install.
